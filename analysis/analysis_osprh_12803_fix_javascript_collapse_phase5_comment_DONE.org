* Understanding Gerrit Topics

** What is a Gerrit Topic?

A "topic" in Gerrit is a label/tag that groups related changes together. Topics are:
- Independent of branch names
- Used to track related work across multiple reviews
- Visible in the review page and searchable
- Optional but highly recommended for organizing work

** Why Topics Matter

Topics help:
1. Group related changes together (e.g., all de-angularization work)
2. Track progress on a specific initiative
3. Review related changes as a set
4. Understand the broader context of a change

** Topic Naming Strategy

For your de-angularization work:

Always use: =de-angularize=

These fragment your work and make it hard to track progress.

** Branch Naming vs Topic Naming

*** Do branch names need to match topics?

*No, they are independent.*

| Concept      | Scope      | Purpose                                 | Example                         |
|--------------+------------+-----------------------------------------+---------------------------------|
| Branch Name  | Local Git  | Organize your local development         | =fix/keypair-chevron=           |
| Topic        | Gerrit     | Group related reviews in Gerrit         | =de-angularize=                 |

*** Recommended Workflow

*Branch Naming (Local):*
- Use descriptive names for what YOU'RE working on
- Example: =fix/keypair-collapsible-rows=
- Example: =osprh-12803-working= (your current branch)
- This helps YOU organize your local work

*Topic Naming (Gerrit):*
- Use broad initiative names
- Example: =de-angularize= for all de-angularization work
- This helps REVIEWERS and the TEAM track related work

*** Example Workflow

#+BEGIN_SRC bash
# Starting work on removing AngularJS from Images panel
cd ~/Work/horizon

# Create local branch with descriptive name
git checkout -b fix/images-panel-deangular

# ... make your changes ...
git add .
git commit -s -m "Remove AngularJS from Images panel

This change de-angularizes the Images panel by converting
to React components and modern JavaScript.

This is part of the broader de-angularization initiative
to remove AngularJS dependencies from Horizon.
"

# Push to Gerrit with the de-angularize topic
git review -t de-angularize
#+END_SRC

Result:
- Local branch: =fix/images-panel-deangular= (just for you)
- Gerrit topic: =de-angularize= (for everyone)

** Setting Topics: When and How

*** For New Reviews

When pushing your first patchset:

#+BEGIN_SRC bash
git review -t de-angularize
#+END_SRC

This sets the topic immediately.

*** For Existing Reviews

If you forgot to set the topic (like with review 966349):
1. Use the Gerrit web UI to edit the topic
2. No need to push a new patchset
3. The topic change is instant and doesn't create a new patchset

*** For Subsequent Patchsets

Once a topic is set on a review, it persists across patchsets.
You don't need to specify it again with =git review=.
