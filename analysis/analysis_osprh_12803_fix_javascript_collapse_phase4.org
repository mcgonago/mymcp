* PEP8 Standards Reference

** Two Blank Lines Rule (E302)

From PEP 8:
#+BEGIN_QUOTE
Separate top-level function and class definitions with two blank lines.

Method definitions inside a class are separated by a single blank line.
#+END_QUOTE

Source: https://peps.python.org/pep-0008/#blank-lines

** No Trailing Whitespace Rule (W293)

From PEP 8:
#+BEGIN_QUOTE
Avoid trailing whitespace anywhere. Because it's usually invisible, it can be
confusing: e.g. a backslash followed by a space and a newline does not count
as a line continuation marker.
#+END_QUOTE

This includes blank lines - they should contain NO whitespace at all.

Source: https://peps.python.org/pep-0008/#whitespace-in-expressions-and-statements

** Line Length Limit (E501)

From PEP 8:
#+BEGIN_QUOTE
Limit all lines to a maximum of 79 characters.

For flowing long blocks of text with fewer structural restrictions (docstrings
or comments), the line length should be limited to 72 characters.
#+END_QUOTE

However, OpenStack generally allows 79 characters for code and docstrings.

Source: https://peps.python.org/pep-0008/#maximum-line-length

** No Blank Line at End of File (W391)

From PEP 8 and Python conventions:
#+BEGIN_QUOTE
Python files should end with a newline character, but not with blank lines.
#+END_QUOTE

The file should end immediately after the last line of code/comment.

** Key Points for This Code

   - Standard library imports
   - Related third party imports
   - Local application/library specific imports
   - Blank line between each group
