** The Issue: Default Link Colors

By default, browsers apply special styling to =<a>= (anchor/link) elements:

- *Default link color*: Usually blue (#0000EE or similar)

- *Hover state*: Often darker blue or underlined
- *Active state*: Often red

1. Makes it stand out visually as a "link" rather than a subtle UI control
2. Doesn't match the color scheme of the rest of the table

* What Does =color: inherit= Mean?

** CSS Inheritance Basics

In CSS, the =color= property can have several values:
- *Specific colors*: =#333333=, =blue=, =rgb(0,0,0)=
- *Keywords*: =inherit=, =initial=, =currentColor=
- *System colors*: =LinkText=, =ButtonFace=, etc.

** The =inherit= Keyword

The =inherit= keyword tells an element:

#+BEGIN_QUOTE
"Don't use your default color. Instead, use whatever color your parent element has."
#+END_QUOTE

** Inheritance Chain Example

#+BEGIN_EXAMPLE
<table style="color: #333;">        ← Color is dark gray (#333)
  <tr>                              ← Inherits #333 from table
    <td>                            ← Inherits #333 from tr
      <a class="chevron-toggle">    ← Has default blue, BUT...
        <span class="fa">▶</span>   ← Would inherit blue from <a>
      </a>
    </td>
  </tr>
</table>

After adding "color: inherit;" to .chevron-toggle:

<table style="color: #333;">        ← Color is dark gray (#333)
  <tr>                              ← Inherits #333 from table
    <td>                            ← Inherits #333 from tr
      <a class="chevron-toggle" 
         style="color: inherit;">   ← NOW inherits #333 from td!
        <span class="fa">▶</span>   ← Inherits #333 from <a>
      </a>
    </td>
  </tr>
</table>
#+END_EXAMPLE

** Technical Definition

From W3C CSS Specification:

#+BEGIN_QUOTE
*inherit*: The property takes the same specified value as the property
for the element's parent. Each property may also have a cascaded value
of 'inherit', which means that, for a given element, the property takes
the same specified value as the property for the element's parent.
#+END_QUOTE

Source: https://www.w3.org/TR/CSS2/cascade.html#value-def-inherit

<style>
/* Chevron rotation for key pair expandable rows
   Uses Bootstrap's automatic .collapsed class management */
.chevron-toggle {
  color: inherit;
  transition: all 0.3s ease;
}

.chevron-toggle .fa-chevron-right {
  display: inline-block;
  transition: transform 0.3s ease;
}

/* Rotate chevron 90 degrees when row is expanded (not collapsed) */
.chevron-toggle:not(.collapsed) .fa-chevron-right {
  transform: rotate(90deg);
}

/* Remove padding from detail row cells to eliminate gap when collapsed */
tr.keypair-detail-row td {
  padding: 0;
}

/* Add margin to inner content to maintain spacing when expanded */
tr.keypair-detail-row td div.keypair-details {
  margin: 8px;
}
</style>
#+END_SRC

** Why This Works

1. The =<a class="chevron-toggle">= is inside a =<td>= (table cell)
2. The =<td>= has the table's default text color (likely #333 or similar gray/black)
3. By adding =color: inherit;=, the =<a>= now uses the =<td>='s color
4. The =<span class="fa fa-chevron-right">= inside the =<a>= inherits from its parent =<a>=
5. Result: The chevron appears in the same color as surrounding text!

** CSS Specificity & Priority

This works because:
- Browser default =<a>= color has *low specificity* (user agent stylesheet)
- Our =.chevron-toggle { color: inherit; }= is an *author stylesheet* with *higher priority*
- Author styles override user agent (browser default) styles

* Why Is This Important?

** Design Consistency

Making the chevron match text color:
1. *Reduces visual noise*: Not every interactive element needs to look like a "link"
2. *Improves focus*: Users focus on the key pair names, not the toggle mechanism
3. *Better UX*: The chevron becomes a subtle, natural part of the table interface

** Horizon Design Patterns

Looking at other Horizon tables with expandable rows or toggle controls:
- They typically use subtle, text-colored icons
- Interactive elements in tables don't always use link colors
- The focus is on data, not UI chrome

** Accessibility Note

Even though we're removing the blue link color:
- The =role="button"= attribute correctly identifies this as an interactive element
- The =aria-label= provides screen reader context
- The cursor changes to pointer on hover (browser default for =<a>=)
- Keyboard navigation still works (=<a>= is naturally focusable)

So removing the blue color doesn't harm accessibility!

* Summary

** What =color: inherit= Means

=color: inherit= tells a CSS element:
- "Don't use your own default color"
- "Use whatever color your parent element has"
- It breaks the normal cascade and explicitly pulls the parent's color value
