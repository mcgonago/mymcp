* Problem Analysis

** Why Custom JavaScript Was Initially Used

In Phase 1, we discovered that Bootstrap's collapse class doesn't work directly 
on ~<tr>~ (table row) elements. This is because:

1. Bootstrap's collapse uses CSS transitions on height
2. Table rows have special display properties (~display: table-row~)
3. CSS height transitions don't work properly on table rows

This led us to implement custom JavaScript to handle show/hide via jQuery's 
~.show()~ and ~.hide()~ methods.

** Bootstrap's Native Collapse Approach

According to Bootstrap 3.4 documentation:
https://getbootstrap.com/docs/3.4/javascript/#collapse

Bootstrap collapse works via data attributes:
- ~data-toggle="collapse"~ - marks the trigger element
- ~data-target="#id"~ (or ~href="#id"~) - specifies the target to collapse
- Target element needs ~class="collapse"~
- Bootstrap automatically handles:
  - Showing/hiding the element
  - Adding/removing ~.in~ class when visible
  - Updating ~aria-expanded~ attribute
  - CSS transitions

** The Key Insight: Collapse the DIV, Not the TR

The solution is to:
1. Keep the ~<tr>~ always visible (remove ~style="display: none;"~)
2. Wrap the CONTENT inside the ~<td>~ with a ~<div class="collapse">~
3. Bootstrap will collapse the DIV inside the cell, not the row itself
4. This approach works because DIVs support CSS height transitions

* How Bootstrap Collapse Works

** Data Attribute API

When Bootstrap's JavaScript is loaded, it scans for elements with 
~data-toggle="collapse"~ and automatically attaches event handlers.

Clicking the trigger:
1. Bootstrap finds the target via ~data-target~ selector
2. Checks if target has ~.in~ class (is currently visible)
3. If not visible:
   - Adds ~.in~ class
   - Animates height from 0 to full height
   - Updates ~aria-expanded="true"~ on trigger
4. If visible:
   - Removes ~.in~ class
   - Animates height from full height to 0
   - Updates ~aria-expanded="false"~ on trigger

** Why This Works Without Custom JavaScript

Bootstrap's collapse plugin:
- Is already loaded in Horizon (Bootstrap is a core dependency)
- Automatically initializes via data attributes
- Handles all the complexity:
  - Height calculations
  - CSS transitions
  - ARIA attribute updates
  - Event management

** Chevron Icon Rotation

The ONE thing Bootstrap doesn't handle automatically is rotating the chevron icon.

We have two options:

*** Option 1: CSS-Only Solution (Cleanest)

Add CSS to automatically rotate the icon based on ~aria-expanded~:

#+BEGIN_SRC css
.chevron-toggle[aria-expanded="true"] .fa-chevron-right {
    transform: rotate(90deg);
    transition: transform 0.3s ease;
}

.chevron-toggle .fa-chevron-right {
    transition: transform 0.3s ease;
}
#+END_SRC

This uses CSS attribute selectors to rotate when ~aria-expanded="true"~.

*** Option 2: Bootstrap Events (More Complex)

Use Bootstrap's collapse events to toggle icon classes:

#+BEGIN_SRC javascript
$('.chevron-toggle').on('click', function() {
    $(this).find('.fa').toggleClass('fa-chevron-right fa-chevron-down');
});
#+END_SRC

*** Recommendation: Option 1 (CSS)

The CSS solution is:
- Cleaner (no JavaScript needed)
- More maintainable
- Uses Bootstrap's own aria-expanded updates
- Follows separation of concerns (presentation in CSS)

However, we need to check if Horizon's CSS architecture allows adding these styles.
If not, Option 2 is a minimal JavaScript solution.

* Potential Issues and Solutions

** Issue 1: Table Row Display
Problem: The ~<tr>~ remains visible even when collapsed.
Solution: This is expected. The ~<div>~ inside the ~<td>~ collapses, making the 
row appear empty. This is actually better UX than hiding the entire row.

** Issue 2: Icon Rotation
Problem: Bootstrap doesn't rotate the chevron icon automatically.
Solution: Use CSS based on ~aria-expanded~ attribute (see Option 1 above).

** Issue 3: Initial State
Problem: Need to ensure rows start collapsed.
Solution: The ~class="collapse"~ without ~.in~ ensures collapsed initial state.

** Issue 4: Horizon Compatibility
Problem: Need to verify Bootstrap version in Horizon.
Solution: Horizon uses Bootstrap 3.x, which supports all these features.
