* The Problem

** Issue 1: The Chevron Column Should Not Be Sortable

Clicking the chevron column header attempts to sort by that column, which doesn't make sense:
- You can't sort rows by a chevron icon
- The chevron is a UI control, not data
- Sorting by this column would be meaningless

Currently, clicking the header might:
- Attempt to sort (will fail or produce unexpected results)
- Confuse users (why is this column sortable?)

** Issue 2: "normal_column" Is Not Semantic

The class name "normal_column" is generic and doesn't describe the purpose:
- Doesn't convey that this is a chevron/toggle column
- Makes CSS targeting less specific
- Harder to style or select in tests

Better class names would be:
- ~chevron_column~ - clearly identifies purpose
- ~expandable_toggle_column~ - descriptive
- ~control_column~ - indicates it's a UI control

** Visual Problem

In the rendered table, the chevron column header might show a sort indicator:

* Solution

** Change 1: Disable Sorting

Add ~sortable=False~ to the column definition:

#+BEGIN_SRC python

#+END_SRC

*Effect:*
- ~self.sortable = False~
- The ~if self.sortable and not self.auto:~ check fails
- ~'sortable'~ class is NOT added
- Result: classes = ~['normal_column']~ (sortable removed)

** Change 2: Add Custom Class

Add ~classes=['chevron_column']~ to the column definition:

#+BEGIN_SRC python
chevron = ExpandableKeyPairColumn("chevron", 
                                   verbose_name="", 
                                   sortable=False,
                                   classes=['chevron_column'])
#+END_SRC

*Effect:*
- ~self.classes~ starts with ~['chevron_column']~
- Metaclass appends ~'normal_column'~ → ~['chevron_column', 'normal_column']~
- ~'sortable'~ is not added (because ~sortable=False~)
- Result: classes = ~['chevron_column', 'normal_column']~

** Wait! Do We Want Both Classes?

Having both ~'chevron_column'~ and ~'normal_column'~ might be fine:
- ~'chevron_column'~ - specific styling for this column
- ~'normal_column'~ - inherits general column styles

But if we want to REPLACE ~'normal_column'~ entirely, we need a different approach.

** Alternative: Override classes After Metaclass

The ~'normal_column'~ is added by the metaclass, so we can't prevent it in the
column constructor. Options:

*** Option A: Live with both classes
- Keep both ~'chevron_column'~ and ~'normal_column'~
- Use ~'chevron_column'~ for specific styles
- Accept ~'normal_column'~ for general styles

*** Option B: Remove in __init__
Override in ~ExpandableKeyPairColumn.__init__()~:

class ExpandableKeyPairColumn(tables.Column):

* Why This Matters

** 1. User Experience

Users should not see sort indicators or sorting behavior on columns that aren't
data columns. The chevron is a control, not content to sort by.

** 2. Semantic HTML

CSS classes should describe the purpose/content of elements:
- ~'normal_column'~ - generic
- ~'chevron_column'~ - specific, meaningful
- ~'sortable'~ - implies behavior that shouldn't exist

** 3. Maintainability

Future developers (or testers) can easily identify and target the chevron column:

- ~multi_select_column~ - for checkbox column

* Alternative Approach: Custom __init__

If we wanted to completely remove ~'normal_column'~, we could override ~__init__~:

#+BEGIN_SRC python
class ExpandableKeyPairColumn(tables.Column):
    """Column that renders a chevron toggle for expandable rows.
    
    This column is not sortable and uses 'chevron_column' class instead
    of the default 'normal_column' class.
    """
    
    def __init__(self, transform, **kwargs):
        # Force sortable=False
        kwargs['sortable'] = False
        # Set custom classes
        kwargs.setdefault('classes', ['chevron_column'])
        # Call parent
        super().__init__(transform, **kwargs)
        # Remove 'normal_column' added by metaclass
        if 'normal_column' in self.classes:
            self.classes.remove('normal_column')
    
    def get_data(self, datum):
        chevron_id = get_chevron_id(self.table, datum)
        return render_to_string(
            "key_pairs/_chevron_column.html",
            {"chevron_id": chevron_id}
        )
#+END_SRC
