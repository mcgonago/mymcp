* Summary: The Answer to "How?"

** Q: Where is "normal_column sortable"?

*A:* In the rendered HTML ~class~ attribute, visible in browser DevTools.
NOT in the Python code!

** Q: How does ~classes=['chevron_column']~ change the class name?

*A:* It ADDS ~'chevron_column'~ to the list. The metaclass still adds 
~'normal_column'~, so the final list is ~['chevron_column', 'normal_column']~.
When joined with spaces, becomes: ~"chevron_column normal_column"~.

** Q: How does ~sortable=False~ prevent sorting?

*A:* Inside ~Column.__init__~, there's a check:
#+BEGIN_SRC python
if self.sortable and not self.auto:
    self.classes.append("sortable")
#+END_SRC

When ~sortable=False~, the condition is False, so "sortable" is never appended.

** Q: How can we add parameters without changing ~ExpandableKeyPairColumn~?

*A:* Python's inheritance! ~ExpandableKeyPairColumn~ has no ~__init__~ method,
so it uses the parent's (~Column.__init__~). We can pass ANY parameter that
~Column.__init__~ accepts, including ~sortable~ and ~classes~.

* Key Takeaway

The class names ~"normal_column sortable"~ are generated by Horizon's framework
and appear in the rendered HTML, NOT in the Python code.

By passing explicit parameters (~sortable=False~ and ~classes=['chevron_column']~),
we control how those class names are generated, ultimately changing what appears
in the HTML ~class~ attribute.
