### Code Review

#### Context: Expandable Rows Feature

This template was added by **Review 966349** to implement expandable rows with chevrons in the Key Pairs table. The expandable row functionality allows users to click a chevron icon to reveal additional details about each key pair.

#### The CSS Change

**Location:** Line 54-55 (after line 53 with `margin-top: -1px;` comment)

```css
.expandable_row_content {
  padding: 0;
  overflow: hidden;
  margin-top: -1px; /* Get rid of the double horizontal line when collapsed */
  border-top: 1px;      /* ← ADDED (INVALID CSS) */
  border-bottom: 1px;   /* ← ADDED (INVALID CSS) */
}
```

#### Why This is Invalid

The CSS `border` shorthand property requires **three values**:
1. **Width:** `1px` ✅ (present)
2. **Style:** `solid`, `dashed`, `dotted`, etc. ❌ (missing)
3. **Color:** `#ddd`, `red`, `rgba(...)`, etc. ❌ (missing)

**Result:** Modern browsers will **ignore** these incomplete declarations. The CSS parser will fail to apply the border, and the "fix" won't actually fix anything.

#### What Probably Should Have Been Done

Based on typical Horizon/Bootstrap patterns:

```css
.expandable_row_content {
  padding: 0;
  overflow: hidden;
  margin-top: -1px; /* Get rid of the double horizontal line when collapsed */
  border-top: 1px solid #ddd;
  border-bottom: 1px solid #ddd;
}
```

Or, to match the existing pattern of negative margins compensating for borders:
```css
.expandable_row_content {
  padding: 0;
  overflow: hidden;
  margin-top: -1px;
  border-top: 1px solid #ddd;
  border-bottom: 1px solid #ddd;
  margin-bottom: -1px; /* Keep symmetry with margin-top */
}
```
